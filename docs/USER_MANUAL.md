## 界面介绍

AI IDE 采用经典的 IDE 布局，分为以下几个区域：

```
┌─────────────────────────────────────────────────────────┐
│ 标题栏 (AI IDE)    [Git] [主题] [语言]                   │  ← 顶部工具栏
├─────────────────────────────────────────────────────────┤
│              │                                          │
│  Git 面板    │         编辑器区域                        │
│  (可折叠)    │         (Monaco Editor)                   │
│              │                                          │
├──────────────┼──────────────────────────────────────────┤
│ 侧边栏       │  活动栏  │                                │
│ (文件浏览器)  │ (快速操作)│                                │
│              │          │                                │
├──────────────┴──────────┴────────────────────────────────┤
│  底部面板 (终端 / AI 对话)                                │
├─────────────────────────────────────────────────────────┤
│  状态栏 (文件数 | 就绪 | Ctrl+Shift+P 命令面板)           │
└─────────────────────────────────────────────────────────┘
```

### 顶部工具栏

位于窗口最上方，包含：

- **应用标题**：显示 "AI IDE"
- **Git 按钮**：点击打开/关闭 Git 面板
- **主题切换按钮**：循环切换 亮色/暗色/跟随系统
- **语言切换按钮**：切换 中文/英文 界面

### Git 面板

位于左侧，显示当前 Git 仓库的状态：

- 当前分支名称
- 文件变更列表（已修改、已添加、未追踪等）
- 点击文件可查看 Diff

### 文件浏览器侧边栏

显示项目文件树：

- 树形目录结构
- 支持展开/折叠目录
- 点击文件可在编辑器中打开

### 活动栏

位于编辑器左侧的窄栏，提供快捷操作按钮：

- 快速打开演示文件
- 预留扩展图标位

### 编辑器区域

核心工作区，基于 Monaco Editor：

- 多标签页编辑
- 语法高亮、自动补全
- 代码折叠、缩略图
- 支持 LSP（跳转到定义、查找引用）

### 底部面板

可切换的两个标签页：

- **终端**：基于 xterm.js 的完整终端，支持 zsh/bash
- **AI 对话**：与 AI 助手进行对话，支持流式输出

### 状态栏

位于窗口最下方，显示：

- 应用名称
- 当前打开文件数量
- 命令面板快捷键提示

### 命令面板

按 `Ctrl+Shift+P`（或 `F1`）打开：

- 搜索所有可用命令
- 显示命令快捷键
- 支持分类筛选

## 基本操作

### 打开项目

1. 点击文件浏览器侧边栏的 "打开文件夹" 按钮（或通过命令面板）
2. 选择项目目录
3. 文件树将自动加载

### 编辑文件

1. 在文件浏览器中点击文件
2. 文件在编辑器中以标签页形式打开
3. 编辑内容后按 `Ctrl+S` 保存
4. 修改的文件标签会显示修改标记

### 使用 AI 助手

1. 点击底部面板的 "AI" 标签
2. 在输入框中输入问题或代码需求
3. 按 Enter 发送
4. AI 将以流式方式回复

AI 支持的功能：
- 代码解释和补全
- Bug 修复建议
- 代码重构
- 通过 MCP 工具搜索代码库、获取网页内容

### 使用终端

1. 点击底部面板的 "终端" 标签（或按 `Ctrl+` `）
2. 终端自动启动默认 Shell（zsh 或 bash）
3. 可直接执行命令，与系统终端体验一致

### Git 操作

1. 打开项目后，Git 面板自动检测仓库状态
2. 查看变更文件列表
3. 点击文件查看 Diff
4. 右键文件可执行暂存/取消暂存
5. 输入提交信息后点击提交按钮

### 切换主题

点击顶部工具栏的主题按钮，循环切换：
- **亮色主题**：适合光线充足的环境
- **暗色主题**：适合夜间或暗光环境
- **跟随系统**：自动匹配操作系统主题

### 切换语言

点击顶部工具栏的语言按钮，在中英文界面间切换。

| 快捷键 | macOS | 功能 |

- macOS 上 `Ctrl` 键自动映射为 `Cmd` 键

- Monaco Editor 内置的快捷键（如多光标编辑、代码格式化）保持默认

## 常见问题

### 应用无法启动

**问题**：运行 `wails dev` 后应用没有启动。

**排查步骤**：
1. 检查 Go 版本是否为 1.23+
2. 检查 Node.js 版本是否为 18+
3. 检查 Wails CLI 是否正确安装：`wails version`
4. 查看终端输出中的错误信息

### 前端热重载不生效

**问题**：修改前端代码后没有自动刷新。

**解决方案**：
1. 确认 `wails dev` 正确启动
2. 检查 `frontend/vite.config.ts` 配置
3. 手动刷新前端页面（如果通过浏览器开发）

### 终端无法输入

**问题**：终端面板显示但无法输入命令。

**排查步骤**：
1. 确认终端标签处于激活状态
2. 检查默认 Shell 是否存在（macOS 默认使用 `/bin/zsh`）
3. 查看后端日志是否有 PTY 启动错误

### AI 对话无响应

**问题**：发送消息后 AI 没有回复。

**排查步骤**：
1. 检查 AI Provider 是否已配置 API 密钥
2. 确认网络连接正常
3. 查看后端日志中的 HTTP 请求错误
4. 尝试切换其他 AI Provider

### Git 面板显示空白

**问题**：打开项目后 Git 面板没有显示任何内容。

**排查步骤**：
1. 确认项目目录是 Git 仓库（包含 `.git` 目录）
2. 检查是否有权限访问 `.git` 目录
3. 尝试在终端中执行 `git status` 看是否正常

### 文件保存失败

**问题**：保存文件时提示错误。

**排查步骤**：
1. 检查文件路径是否包含非法字符
2. 确认对目标目录有写入权限
3. 检查磁盘空间是否充足

### 主题切换不生效

**问题**：切换主题后界面颜色没有变化。

**解决方案**：
1. 检查 `useThemeStore` 是否正确初始化
2. 确认 Tailwind CSS 的 dark mode 配置正确
3. 尝试重启应用

### 构建产物体积过大

**问题**：`wails build` 生成的应用体积超出预期。

**优化建议**：
1. 使用 `wails build -ldflags="-s -w"` 去除调试信息
2. 检查前端依赖，移除未使用的包
3. 使用 `upx` 压缩可执行文件（可选）

### 跨平台构建失败

**问题**：在非目标平台上构建失败。

**解决方案**：
1. macOS 应用必须在 macOS 上构建（或配置交叉编译 SDK）
2. Windows 应用需要 Windows 上的 CGO 环境
3. 建议使用 GitHub Actions CI 进行多平台构建
