# AI IDE 桌面端手动验证清单

本清单涵盖无法通过 React Testing Library 自动化的桌面 GUI 交互验证项。
请在每次发布前逐项执行并打勾确认。

---

## 1. FileSystem（文件系统）

| 编号 | 检查项 | 预期结果 | 测试步骤 |
|------|--------|----------|----------|
| FS-01 | 打开项目文件夹 | 文件浏览器显示项目目录树 | 1. 点击菜单「文件」→「打开文件夹」<br>2. 选择一个包含代码文件的项目目录<br>3. 确认左侧文件树正确加载 |
| FS-02 | 文件树展开/折叠 | 目录展开显示子文件，折叠隐藏 | 1. 点击含子目录的文件夹旁的展开图标<br>2. 确认子文件/目录显示<br>3. 再次点击折叠图标，确认子项隐藏 |
| FS-03 | 新建文件 | 新文件出现在文件树中并自动打开 | 1. 右键点击目录 →「新建文件」<br>2. 输入文件名 `test.ts`<br>3. 确认文件出现在目录下且编辑器打开新标签 |
| FS-04 | 新建文件夹 | 新文件夹出现在文件树中 | 1. 右键点击目录 →「新建文件夹」<br>2. 输入文件夹名 `new-folder`<br>3. 确认文件夹出现在目录下 |
| FS-05 | 重命名文件 | 文件名更新，已打开的标签同步更新 | 1. 右键点击文件 →「重命名」<br>2. 修改文件名<br>3. 确认文件树和标签页标题同步变更 |
| FS-06 | 删除文件 | 文件从文件树消失，标签页关闭 | 1. 右键点击文件 →「删除」<br>2. 确认弹窗后点击确定<br>3. 确认文件树和标签页同步移除 |
| FS-07 | 文件变更监听 | 外部修改的文件在树中高亮 | 1. 在系统文件管理器中修改项目中的某个文件<br>2. 返回 IDE，确认文件在树中显示 Git 变更指示器 |
| FS-08 | 拖拽文件到编辑器 | 文件在编辑器中打开 | 1. 从系统文件管理器拖拽一个代码文件到 IDE 编辑器区域<br>2. 确认文件以新标签页形式打开 |

## 2. Editor（编辑器）

| 编号 | 检查项 | 预期结果 | 测试步骤 |
|------|--------|----------|----------|
| ED-01 | Monaco 编辑器加载 | 代码文件在 Monaco 中正确渲染，带语法高亮 | 1. 打开任意 `.tsx` 文件<br>2. 确认 Monaco 编辑器加载成功，代码带颜色高亮 |
| ED-02 | 代码自动补全 | 输入时显示 IntelliSense 建议列表 | 1. 在 TypeScript 文件中输入 `console.`<br>2. 确认弹出自动补全列表，包含 `log`、`error` 等方法 |
| ED-03 | 跳转到定义 | 光标跳转到符号定义位置 | 1. 按住 `Cmd/Ctrl` 键并点击函数调用<br>2. 确认编辑器跳转到该函数的定义处 |
| ED-04 | 查找引用 | 显示所有引用该符号的位置 | 1. 右键点击函数名 →「查找所有引用」<br>2. 确认底部面板列出所有引用位置 |
| ED-05 | 代码折叠 | 代码块可折叠/展开 | 1. 点击行号旁的折叠图标（减号）<br>2. 确认代码块隐藏，图标变为加号<br>3. 再次点击展开 |
| ED-06 | 多光标编辑 | 多个光标同时输入 | 1. 按住 `Alt/Option` 点击多处位置<br>2. 输入文字，确认所有光标位置同步输入 |
| ED-07 | 缩略图(Minimap) | 右侧显示代码缩略图 | 1. 打开一个超过 50 行的文件<br>2. 确认编辑器右侧出现缩略图<br>3. 在缩略图上点击可跳转 |
| ED-08 | 自动保存 | 文件在失焦后自动保存 | 1. 在设置中开启「自动保存」<br>2. 修改文件内容后点击其他标签<br>3. 确认文件标题无未保存标记 |
| ED-09 | 保存时格式化 | 保存后代码自动格式化 | 1. 在设置中开启「保存时格式化」<br>2. 故意将代码缩进打乱<br>3. 按 `Ctrl+S` 保存，确认代码恢复正确缩进 |
| ED-10 | Diff 预览 | 显示文件修改前后的差异对比 | 1. 在 Git 面板中点击一个已修改文件<br>2. 确认弹出 Diff 面板，正确显示增删行 |

## 3. Git（版本控制）

| 编号 | 检查项 | 预期结果 | 测试步骤 |
|------|--------|----------|----------|
| GT-01 | Git 状态检测 | 打开 Git 项目时面板显示分支和文件状态 | 1. 打开一个 Git 仓库项目<br>2. 确认 Git 面板显示当前分支名和文件变更列表 |
| GT-02 | 暂存文件 | 文件从「已修改」移动到「已暂存」 | 1. 修改一个文件并保存<br>2. 在 Git 面板点击该文件旁的暂存按钮（→）<br>3. 确认文件移动到「已暂存」区域 |
| GT-03 | 取消暂存 | 文件从「已暂存」移回「已修改」 | 1. 在「已暂存」区域点击文件的取消暂存按钮（✓）<br>2. 确认文件移回「已修改」区域 |
| GT-04 | 提交更改 | 提交成功，工作区变干净 | 1. 暂存若干文件<br>2. 在提交信息框中输入提交信息<br>3. 点击「提交」按钮<br>4. 确认工作区显示「干净」状态 |
| GT-05 | 推送代码 | 代码成功推送到远程仓库 | 1. 进行一次提交<br>2. 点击 Git 面板顶部的推送按钮（↑）<br>3. 确认 ahead 计数归零 |
| GT-06 | 拉取代码 | 远程更新成功合并到本地 | 1. 在远程仓库提交新更改<br>2. 点击 Git 面板顶部的拉取按钮（↓）<br>3. 确认文件树和编辑器反映最新内容 |
| GT-07 | 查看 Diff | 正确显示文件的增删差异 | 1. 修改一个文件<br>2. 在 Git 面板点击该文件名<br>3. 确认弹出 Diff 视图，绿色为新增，红色为删除 |
| GT-08 | 分支切换 | 成功切换到目标分支，文件更新 | 1. 在 Git 面板查看当前分支<br>2. 通过后端 API 或其他方式切换分支<br>3. 确认 Git 面板刷新显示新分支名和对应文件状态 |

## 4. Terminal（终端）

| 编号 | 检查项 | 预期结果 | 测试步骤 |
|------|--------|----------|----------|
| TM-01 | 终端面板打开 | xterm.js 终端渲染成功 | 1. 按 `Ctrl+\`` 或点击菜单打开终端<br>2. 确认底部出现终端面板，显示 Shell 提示符 |
| TM-02 | 终端命令执行 | 命令正确执行并显示输出 | 1. 在终端中输入 `echo "hello"`<br>2. 按回车<br>3. 确认终端显示 `hello` |
| TM-03 | PTY 交互 | 支持交互式命令（如 vim、top） | 1. 在终端中输入 `vim test.txt`<br>2. 确认 vim 界面正确渲染<br>3. 按 `:q` 退出 |
| TM-04 | 终端字体大小调整 | 字体大小随设置变更 | 1. 打开设置 → 终端 → 字体大小<br>2. 修改字体大小为 18<br>3. 确认终端字体同步变大 |
| TM-05 | 终端光标样式 | 光标样式随设置变更 | 1. 打开设置 → 终端 → 光标样式<br>2. 切换为「竖线」<br>3. 确认终端光标变为竖线形 |
| TM-06 | 终端回滚 | 可滚动查看历史输出 | 1. 在终端中执行 `seq 1 100`<br>2. 向上滚动<br>3. 确认可以查看历史输出内容 |
| TM-07 | 多终端标签 | 支持打开多个终端实例 | 1. 点击终端面板旁的「+」按钮<br>2. 确认新增终端标签<br>3. 在不同标签中执行不同命令，确认互不干扰 |
| TM-08 | 终端复制粘贴 | 支持选中文本复制和粘贴 | 1. 在终端中选中一段文字<br>2. 按 `Cmd/Ctrl+C` 复制<br>3. 按 `Cmd/Ctrl+V` 粘贴到终端输入行 |

## 5. AI（AI 助手）

| 编号 | 检查项 | 预期结果 | 测试步骤 |
|------|--------|----------|----------|
| AI-01 | AI 面板打开 | 聊天界面正确渲染 | 1. 点击左侧 AI 图标或按快捷键打开 AI 面板<br>2. 确认聊天输入框和历史记录区域显示正常 |
| AI-02 | 流式对话 | AI 回复以流式方式逐字显示 | 1. 在 AI 输入框中输入「写一个快速排序算法」<br>2. 发送消息<br>3. 确认 AI 回复以打字机效果逐字出现 |
| AI-03 | 代码块渲染 | AI 回复中的代码带语法高亮 | 1. 请求 AI 生成一段 TypeScript 代码<br>2. 确认回复中的代码块带语法高亮和复制按钮 |
| AI-04 | 插入代码到编辑器 | 代码正确插入当前光标位置 | 1. 在 AI 回复中找到代码块<br>2. 点击「插入编辑器」按钮<br>3. 确认代码插入到当前激活编辑器的光标位置 |

| AI-06 | MCP 工具调用 | AI 可调用代码搜索等工具 | 1. 询问 AI「项目中哪里用了 useEffect」<br>2. 确认 AI 调用代码搜索工具并返回结果 |
| AI-07 | 对话历史持久化 | 重启后历史记录保留 | 1. 进行若干轮对话<br>2. 关闭并重新打开 IDE<br>3. 确认 AI 面板保留之前的对话历史 |
| AI-08 | 清空对话 | 对话历史被清空 | 1. 点击 AI 面板的「清空」按钮<br>2. 确认对话历史被清空，仅剩系统提示 |

## 6. LSP（语言服务器协议）

| 编号 | 检查项 | 预期结果 | 测试步骤 |
|------|--------|----------|----------|
| LS-01 | LSP 启动 | 打开对应语言文件时 LSP 启动 | 1. 打开一个 `.go` 或 `.ts` 文件<br>2. 查看底部状态栏，确认 LSP 连接状态显示为「已连接」 |
| LS-02 | 实时诊断 | 代码错误显示红色波浪线 | 1. 故意在 TypeScript 文件中写一个类型错误（如 `const x: number = "str"`）<br>2. 确认错误处显示红色波浪下划线 |
| LS-03 | 悬停提示 | 鼠标悬停显示类型信息 | 1. 将鼠标悬停在变量或函数名上<br>2. 确认弹出提示框显示类型签名和文档 |
| LS-04 | 格式化文档 | 整个文档按 LSP 规则格式化 | 1. 故意打乱代码缩进和换行<br>2. 右键菜单选择「格式化文档」或按快捷键<br>3. 确认代码恢复规范格式 |
| LS-05 | 重命名符号 | 所有引用同步更新 | 1. 右键点击函数名 →「重命名符号」<br>2. 输入新名称<br>3. 确认该函数所有调用处同步更新 |
| LS-06 | 代码操作 | 显示可用的代码修复建议 | 1. 在 TypeScript 中写一个未使用的导入<br>2. 点击灯泡图标或按快捷键<br>3. 确认显示「移除未使用的导入」等建议 |
| LS-07 | 自动导入 | 自动添加缺失的导入语句 | 1. 输入一个未导入的符号（如 `useState`）<br>2. 从自动补全列表中选择<br>3. 确认文件顶部自动添加 `import { useState } from 'react'` |
